# 4주차 실습정리

## 오늘 실습 한 것
- 원형 / AABB / OBB Bounding Box 시각화
- 세 방식의 충돌 판정 차이 관찰

## 관찰 내용

### 원형 vs AABB
- AABB에 원형이 먼저 충돌 했을때 아무 일도 일어나지 않았다.
- 둘이 동시에 충돌했을때 화면이 노란색으로 변화 하였다.

### AABB vs OBB
- 모서리로 접근을 하였을떄 AABB만 HIT하였다.
- 한쪽면으로 접근 하였을때 3가지 방식 모두 HIT하였다.
- 회전을 빠르게 하며 접근 하였을때 AABB는 개속 HIT상태를 유지하였고 OBB는 두 오브잭트가 겹쳤을때만 HIT했다.

## AI질문을 하면서 성찰할 것
- 두개의 오브잭트의 이미지를 우주선 모양으로 바꾸기위해 프롬프트 작성(실패)

### 왜 실패 했는가
- 프롬프트가 너무 빈약함(그냥 코드만 복사해서 합쳐달라 작성)
- 문제점을 보고도 왜 싫패 했는지 원인을 알아보지 않고 개속 다시 작성 해달라고만함.

### 성찰
- 메모장에 프롬프트를 미리작성(1~2줄이 아닌 4줄이상 또는 한줄식 여러번)
- 싫패 했다면 다시해달라고 하기전에 뭐가 문제인지 먼저 물어본다.

## 나의 게임에 쓴다면
- 나는 OBB방식을 쓸것이다. / 게임을 하다보면 이게 맞는 판정인가? 하는 상황자주 격어 만약 내가 게이을 만든다면 정확한 판정으로 게임을 만들겠다는 다짐을 했어서, AABB 보단 OBB가 더 적합해보인다.
